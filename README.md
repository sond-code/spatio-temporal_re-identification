# Spatio-Temporal Vehicle Re-Identification System

## 📌 Overview
This project implements a **Vehicle Re-Identification (ReID)** system using the **VeRi dataset**, combining:

- Appearance-based matching using **TransReID (Vision Transformer)**
- Fast similarity search using **FAISS**
- Spatial-temporal reasoning using camera topology
- Multi-feature fusion using a trained **FusionNet**
- Interactive visualization via **Streamlit**

The system retrieves similar vehicles across cameras and improves ranking using temporal and contextual constraints.

### 4. What Each File Does

#### Main application
1- demo_app.py

This is the main entry point of the project.
It launches the interactive demo, typically through Streamlit, and allows the user to:

- select a query image
- retrieve candidate matches from the gallery
- compare embeddings
- apply appearance and temporal scoring
- use FusionNet to produce a final ranking
- visualize retrieved images and scores

---

#### Dataset and metadata
2- data/VeRi/

Copy of the VeRi dataset used by the project.

It contains:

> image_train/: training split
> image_test/: gallery / test split
> image_query/: query split

These images are used for:

- model training
- embedding generation
- retrieval and demo visualization


3- camera_Dist.txt

Distance information between cameras in the VeRi environment.
Used for spatial reasoning and temporal plausibility between camera transitions.


4- veri_partial_visible_plates.csv

Contains license-plate-related metadata for some vehicles.
This can be used as an additional signal or inspection feature in the retrieval pipeline.


5- veri_topology.npz

Stores the learned or generated topology / transition information between cameras.
Used to estimate how likely a vehicle transition is from one camera to another over time.


6- YongtaiPoint_Google.jpg

Reference image for the camera environment / map visualization.
Used only for display and interpretation in the demo.

---

#### Embeddings and retrieval files
7- image_embeddings.csv

File containing exported embeddings for images.
Each row usually includes:

- image name
- vehicle ID
- camera ID
- frame ID
- embedding vector

These embeddings are generated by the trained TransReID model.


8- vehicle_embeddings.index

FAISS index built from the embeddings.
This allows fast nearest-neighbor retrieval instead of comparing all embeddings one by one.


9- vehicle_embeddings_meta.pkl

Metadata file associated with the FAISS index.
It typically maps FAISS entries back to:

- image names
- IDs
- camera info
- frame numbers
- other metadata needed for ranking and display


---

#### Model files
10- models/fusionNet-model/
 fusionnet.pth

Saved trained FusionNet weights.

FusionNet is responsible for combining multiple scores/features, such as:

- appearance similarity
- temporal score
- topology constraints
- optional contextual metadata

to produce a more reliable final score than appearance alone.


11- train_fusionnet_from_pairs.py

Training script used to train FusionNet from generated training pairs / candidate matches.



12- models/transReId-model/
config/VeRi/veri_vit_base_no_jpm.yml

Main configuration file used to train/evaluate the TransReID model on VeRi.



13- pretrain/jx_vit_base_p16_224-80ecf9dd.pth

ImageNet pretrained ViT weights used as the initialization for TransReID.



14- veri_vit_base_no_jpm/train_log.txt

Training log of the TransReID model.



15- veri_vit_base_no_jpm/transformer_120.pth

Contains model checkpoint saved after training.

The checkpoint is used for:

- inference
- embedding extraction
- evaluation
- demo retrieval

---

#### Utility scripts
16- utils/export_image_embeddings_transreid.py

Uses the trained TransReID model to generate embeddings for dataset images.


17- utils/build_faiss_index.py

Builds the FAISS index from the exported embeddings.


18- utils/build_topology_from_train_names.py

Builds or derives topology / temporal transition information based on the training data and camera ordering.


19- utils/build_veri_metadata.py

Parses dataset file names and constructs metadata such as:

- vehicle ID
- camera ID
- frame ID
- split information


20- utils/db_utils.py

Helper functions for loading data files, querying metadata, and supporting the demo pipeline.

## Pipeline Overview

The system works in the following stages:

- Input query image
- Extract appearance embedding using the trained TransReID model
- Search nearest neighbors in the FAISS index
- Load metadata for retrieved candidates
- Compute temporal / topology score
- Fuse appearance + temporal/contextual features using FusionNet
- Return final ranked matches
- Display results in the demo app

## Rebuilding Files (Optional)

If the prebuilt files are missing, the pipeline can be rebuilt in stages.

1. Export embeddings
python utils/export_image_embeddings_transreid.py
2. Build FAISS index
python utils/build_faiss_index.py
3. Build metadata / topology if needed
python utils/build_veri_metadata.py
python utils/build_topology_from_train_names.py
4. Run the demo
streamlit run demo_app.py
